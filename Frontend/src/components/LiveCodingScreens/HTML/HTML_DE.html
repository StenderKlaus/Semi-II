HTML Tags and Functions

Basic HTML

DOCTYPE Declaration (<!DOCTYPE>)

Hiermit wird der Dokumenten-Typ "html" festgelegt

Hier wird mit "en" die verwendete Sprache der Webseite beschrieben ("en" = Englisch, oder "de" = Deutsch). Ausserdem beginnt hier für Browser der mit HTML-Code definierte Bereich.

Example:

<!DOCTYPE html>

HTML Element (<html>)

Der <html> element is the root element of an HTML document. It contains all the other elements on the page.

Example:

<html>
  <head>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

Head Element (<head>)

<head> Im sogenannten "Kopf-Bereich" werden für diese Seite alle allgemein gültigen Links, Scripte, der Seiten-Titel usw. festgelegt. Alle hier hinterlegten Vorgaben sind auf der Webseite nicht zu sehen..

Example:

<head>
  <title>Document Title</title>
</head>

Title Element (<title>)

<title> In jedem Browser kann ganz oben der "Seitentitel" angezeigt werden. Dieser Titel wird hier definiert

Example:

<title>Document Title</title>

Body Element (<body>)

<body> Der "Content-Bereich" beginnt hier. Alles was ab hier eingegeben wird, kann auf den Ausgabemedien dargestellt werden.

Example:

<body>
  <h1>Hallo, Welt!</h1>
</body>

Heading Tags (<h1> to <h6>)

<h1>, <h2>, <h3>, <h4>, <h5>, <h6> erzeugen Überschriften in HTML-Seiten. <h1> steht für die wichtigste Überschrift, gefolgt von <h2>, <h3> usw.. Ohne CSS stellen die <h1>- bis <h6>-Header einen Text in abgestufen Größen dar, wobei <h1> die größte und <h6> die kleinste Schrift benutzt.

Die Überschriften bilden hierarchische Bereiche und strukturieren die Seite – für den Leser, die Suchmaschinen und Screenreader.

Inhalt logisch strukturieren:

Mit den alten HTML-Versionen waren Überschriften das einzige Mittel, um den Inhalt von Webseiten logisch zu strukturieren.

Für den Leser soll die Struktur des Inhalts durch Größe und Farbe der Überschriften klar erkennbar sein; für Screenreader und Suchmaschinen anhand der Hierarchie der <h1>, <h2>, <h3>-Tags.

Example:

<h1>Main Heading</h1>
<h2>Subheading</h2>

Paragraphs (<p>)

Das HTML- Element <p> repräsentiert einen Absatz. Absätze werden in visuellen Medien normalerweise als Textblöcke dargestellt, die durch Leerzeilen und/oder Einrückung in der ersten Zeile von benachbarten Blöcken getrennt sind, aber HTML-Absätze können jede strukturelle Gruppierung verwandter Inhalte sein, wie z. B. Bilder oder Formularfelder.

Absätze sind Elemente auf Blockebene und werden insbesondere automatisch geschlossen, wenn ein anderes Element auf Blockebene vor dem schließenden </p> -Tag geparst wird.

Example:

<p>This is a paragraph.</p>

Line Break (<br>)

HTML <br> erzwingt einen Zeilenumbruch oder -vorschub und ist äquivalent zu einem Linefeed (Zeilenvorschub, in Textverarbeitungsprogrammen meist durch Shift-Return bzw. Umschalt+Eingabetaste). Anders als das Beenden und Öffnen eines neuen p-Tags führt das <br>-Tag nicht zu einer Änderung des Absatzformats.

Example:

This is the first line.<br>This is the second line.

Horizontal Line (<hr>)

Die Browser stellen <hr> mit 100% Breite, 2 Pixel Höhe als schwarze Linie und einem leichten 3D-Effekt dar. Der Dokumentenfluss wird unterbrochen, denn das <hr>-Tag erzeugt einen Zeilenumbruch. Der Linie folgt erneut ein Zeilenumbruch und der Dokumentenfluss wird wieder aufgenommen. Die hr-Linie wirkt einen Hauch eingedrückt, ist schlicht und grau und läuft ohne spezielles CSS über die gesamte Breite der umfassenden Box. Der exakte Abstand zwischen dem Text oder den Bildern von der Linie hängt vom jeweiligen Browser ab oder wird vom CSS gesteuert.

Example:

<p>This is some text.</p>
<hr>
<p>This is some more text.</p>

Comment (<!--...-->)

Kommentare werden verwendet, um dem HTML-Code Anmerkungen oder Erklärungen hinzuzufügen. Sie werden nicht auf der Webseite angezeigt und werden vom Browser ignoriert.

Example:

<!-- This is a comment -->
<p>This is some text.</p>

Forms and Input

Textarea

Der <textarea> Tag wird verwendet, um einen mehrzeiligen Texteingabebereich zu erstellen.

Example:

<textarea rows="4" cols="50">Enter text here...</textarea>

Button

Das <Button> HTML-Element ist ein interaktives Element, das von einem Benutzer mit einer Maus, einer Tastatur, Finger, Sprachbefehl oder einer anderen Hilfstechnologie aktiviert wird. Sobald es aktiviert ist, führt es eine programmierbare Aktion aus, wie zum Beispiel das Absenden eines Formulars oder das Öffnen eines Dialogs. Standardmäßig werden HTML-Schaltflächen in einem Stil dargestellt, der der Plattform ähnelt, auf der der User Agent läuft. Sie können jedoch das Aussehen der Schaltflächen mit CSS ändern.

Example:

<button type="button">Click Me</button>

Select

Das HTML <select>-Element erzeugt eine Drop-Down-Liste in einem Formular.

Per Vorgabe erzeugt <select> ein Fenster mit ausklappenden Optionen. Ein <select> spart Platz im Formular und zeigt erst bei einem Klick die Liste der Optionen an und beschränkt die Anwahl auf die angebotenen Begriffe.

Optional zeigt das <select>-Element auch mehrere Zeilen / Optionen gleichzeitig an. Wenn der Besucher das <select>-Element anklickt, öffnet sich die Liste und klappt alle Auswahlmöglichkeiten auf. Der Besucher wählt einen Eintrag durch einen Klick mit der Maus oder mit einem Fingertippen auf Touchscreens. Optional kann der Besucher auch mehrere Auswahlen treffen.

Example:

<select name="cars"> <option value="volvo">Volvo</option> <option value="bmw">BMW</option> <option value="audi">Audi</option> </select>

Optgroup

Der <optgroup> Tag wird verwendet, um verwandte Optionen in einer Dropdown-Liste zu gruppieren.

Example:

<select name="cars"> <optgroup label="Swedish Cars"> <option value="volvo">Volvo</option> <option value="saab">Saab</option> </optgroup> <optgroup label="German Cars"> <option value="audi">Audi</option> <option value="bmw">BMW</option> </optgroup> </select>

Option

Der <option> Tag wird verwendet, um eine Option in einer Dropdown-Liste zu definieren.

Example:

<select name="cars"> <option value="volvo">Volvo</option> <option value="bmw" selected>BMW</option> <option value="audi">Audi</option> </select>

Label

HTML <label> verknüpft Eingabefelder, Button und Checkboxen eines Formulars mit der Beschriftung des Eingabefelds. Das <for>-Attribut im <label> muss mit der id des Eingabefelds übereinstimmen.

<label> reduziert das Markup, verbessert die Barrierefreiheit des Formulars und lässt sich mit CSS stylen.

<label> for verbindet das <label>-Tag mit einem Eingabeelement des Formulars. Das <for>-Attribut des <label>-Tags muss denselben Wert wie das id-Attribut des Eingabefelds (<input>, <textarea>, <button>, <select>) aufweisen.

Jedes Eingabefeld kann nur ein <label>-Tag haben.

Wenn der Text für die Beschreibung des Eingabefeldes – z.B. "Ihr Name" – in ein <label>-Tag gesetzt wird, entstehen keine sichtbaren Änderungen. Ohne CSS-Stile hat das <label>-Tag keinen Einfluss auf die Gestaltung.

Example:

<label for="name">Name:</label> <input type="text" id="name" name="name">

Fieldset

Der <fieldset> Tag wird verwendet, um verwandte Formularelemente zusammenzufassen.

Example:

<fieldset> <legend>Contact Information</legend> <label for="name">Name:</label> <input type="text" id="name" name="name"> <label for="email">Email:</label> <input type="email" id="email" name="email"> </fieldset>

Legend

Der <legend> Tag wird genutzt um ein Element im <fieldset> zu definieren.

Example:

<fieldset> <legend>Contact Information</legend> <label for="name">Name:</label> <input type="text" id="name" name="name"> <label for="email">Email:</label> <input type="email" id="email" name="email"> </fieldset>

Datalist

Der <datalist> Tag wird verwendet, um eine vordefinierte Liste von Optionen für ein Eingabeelement bereitzustellen.

Example:

<input list="browsers"> <datalist id="browsers"> <option value="Chrome"> <option value="Firefox"> <option value="Safari"> <option value="Opera"> <option value="Edge"> </datalist>

Output

Der <output> Tag wird verwendet, um das Ergebnis einer Berechnung oder einer Benutzeraktion anzuzeigen.

Example:

<form oninput="result.value=parseInt(a.value)+parseInt(b.value)"> <input type="number" id="a" name="a"> + <input type="number" id="b" name="b"> = <output name="result" for="a b"></output> <input type="submit" value="Calculate"> </form>

Frames and iframes

Der <frame> Tag definiert einen individuellen Rahmen in einem "Übergeordneten Rahmen". Er wird benutzt in Verbindung mit <frameset> und <noframes> Tags.

Example:

<frameset cols="25%,75%"> <frame src="menu.html"> <frame src="content.html"> <noframes> <p>This page requires a frames-enabled browser.</p> </noframes> </frameset>

Der <iframe> Tag wird verwendet, um ein anderes HTML-Dokument in das aktuelle Dokument einzubetten. Es ermöglicht Ihnen, Inhalte aus einer anderen Quelle anzuzeigen.

Example:

<iframe src="https://youtu.be/Htq2EUgcrKE"></iframe>

Images

Das Tag <img> wird in HTML für die Beschreibung von Bildern verwendet. Das Bild selbst wird im Dokument nicht direkt platziert, der Browser lädt es aus seiner Quelle, die im Tag <img> eingegeben ist. Für die Eingabe der Quelle wird das Attribut src verwendet. Es gibt zwei erforderliche Attribute für das Element <img>: src, das zur Anzeige der Bildquelle verwendet wird, und alt, das einen alternativen Text für das Bild definiert. Um HTML-Bilder klickbar zu machen, sollen Sie das Tag <img> im Tag <a> platzieren. Das Tag <a> wird zum Einfügen von Links verwendet.

Example:

<img src="image.jpg" alt="Description of the image">

Image Map

HTML <area> und <map> teilen ein Bild in eine Area Map oder Image Map. Area Maps bestehen aus Bildern, über denen maus-sensitive Bereiche liegen. Diese Bereiche verzweigen mit Links zu anderen Seiten oder lösen ein Script aus, um die Area Map zu animieren. HTML Area Maps oder Image Maps waren Knüller, als kleine GIF-Animationen die frühen Internetnutzer noch begeisterten. Die Image-Map ist ein Bild, das an mehreren Stellen auf Aktionen des Benutzers antwortet. Das <map>-Tag verweist auf ein Bild und die darunter liegenden <area>-Tags unterteilt das Bild in Regionen, die von der Maus oder durch ein Touch angesprochen werden. Die <area>-Elemente innerhalb des <map>-Tags legen die Bereiche durch Formen (shape) fest.

Example:

<img src="image.jpg" alt="Image Map" usemap="#image-map"> <map name="image-map"> <area shape="rect" coords="0,0,100,100" href="page1.html" alt="Area 1"> <area shape="circle" coords="150,150,50" href="page2.html" alt="Area 2"> </map>

Canvas

Das englische Wort Canvas bedeutet auf deutsch Leinwand und ist ziemlich genau das: Eine Leinwand, auf die man mit Javascript dynamisch Bitmap-Grafiken malen kann. Erfunden wurde das Element vom Webkit-Team und später sind auch Mozilla und Opera auf den <canvas>-Zug aufgesprungen. Damit ist das das Element heutzutage in alle anständigen Browser implementiert (für den IE gibt es einen Trick – dazu später mehr) und Teil des werdenden HTML5-Standards. Die Möglichkeiten von <canvas> sind relativ weit gefasst. Wann immer man man dynamisch Grafiken erzeugen möchte, kann man das mit <canvas> machen. Beispiele sind z.B. die bekannte Image-Replace-Technik "Typeface.js"

Example:

<canvas id="myCanvas" width="200" height="200"></canvas>

Figure and Figcaption

Bis HTML5 gab es kein besonderes Markup für Abbildungen und die Unterschriften von Bildern oder Grafiken. Während in Zeitschriften oder Büchern kaum ein Bild oder eine Abbildung ohne Unterschrift eingesetzt wird, gab es für die Bildunterschrift in HTML kein semantisches Markup im Standard.

HTML <figure> umschließt eine oder mehrere Abbildungen im weitesten Sinne. Das <figure>-Tag ist keinesfalls auf Bilder oder Grafiken beschränkt. Es kann ebensogut SVG- oder Math-Objekte, Tabellen oder Videos enthalten – am besten lässt sich der Einsatz mit Abbildungen in Fachbüchern vergleichen. Dabei soll aber nicht jedes Bild in ein <figure>-Tag – beim Logo einer Seite z.B. macht ein <figure>-Tag keinen Sinn.

<figure> ersetzt das allgegenwärtige div-Tag als Gruppe von zusammengehörenden Elementen, die als Ergänzung der Seite hervorgehoben werden und nicht an einer bestimmten Stelle im Textfluss liegen müssen – zu vergleichen mit einem Abbildungskasten in einem Fachbuch.

Example:

<figure> <img src="image.jpg" alt="Description of the image"> <figcaption>Caption for the image</figcaption> </figure>

Pictures and Source Sets

Das HTML <picture>-Element liefert zusammen mit srcset und media verschiedene Dateiformate und Seitenverhältnisse an den Browser des Benutzers, z.B. ein quadratisches Bild für kleine Monitore im Hochformat und 16:9 für Desktop-Monitore. Als Konditionen kommen die Auflösung des Monitors, die Größe des Viewports oder die Ausrichtung des Geräts in Frage.

Diese Technik wird auch als Art Direction bezeichnet.

Example:

<picture> <source media="(min-width: 768px)" srcset="large-image.jpg"> <source media="(min-width: 480px)" srcset="medium-image.jpg"> <img src="small-image.jpg" alt="Description of the image"> </picture>

SVG

Der <svg> Tag wird verwendet, um skalierbare Vektorgrafiken in eine HTML-Seite einzubetten.

Example:

<svg width="100" height="100"> <circle cx="50" cy="50" r="40" fill="red" /> </svg>

Audio / Video

Hiermit können Audio Streams oder Sound genutzt werden.

Das HTML

src = Der Pfad und Dateiname des Sounds angegeben werden

autoplay = Wenn der Wert “autoplay” eingetragen ist, dann startet die Sound Datei nach dem laden der Datei

controls = Bei eingetragenen Wert “controls” werden die Steuerelemente angezeigt

loop = Wenn der Parameter “loop” eingetragen ist, dann wird die Musik endlos wiederholt

preload = Wenn “preload” eingetragen wird, dann wird der Sound Datei zusammen mit der Webseite geladen. Dies funktioniert nur, wenn das Attribut autoplay nicht genutzt wird!

muted = Schaltet die Wiedergabe auf Lautlos

crossorigin = Festlegen, wie das Steuerelement ursprungsfremde Anfragen behandelt

Example:

<audio src="audio.mp3" controls> </audio>

Source

Das Tag <source> bestimmt einige multimediale Dateien in verschiedenen Formaten wie Audio und Video. Es ist für die Eigenschaft Cross-Browser notwendig: Der Browser kann aus zugänglichen Varianten das Format, das er unterstützt und ohne Probleme Audio- oder Videodateien abspielt, wählen. In einem Dokument kann das Element <source> mehrmals für die Eingabe von alternativen Audio- und Videodateinen und die Eingabe von einigen Formaten verwendet werden.

Example:

<audio controls> <source src="audio.mp3" type="audio/mpeg"> <source src="audio.ogg" type="audio/ogg"> </audio>

Track

Der <track> Tag wird als sogenanntes "Child" des <audio> Tags oder <video> Tag um zeitgesteuerte Textspuren, wie z. B. Bildunterschriften oder Untertitel, anzugeben.

Example:

<video controls> <source src="video.mp4" type="video/mp4"> <track src="captions.vtt" kind="captions" label="English" srclang="en"> </video>

Video

Mit HTML5 können Sie Filme ohne Plugins oder Add-Ons wiedergeben. Dabei referenziert das video-Element als Container-Element auf eine (oder mehrere) Videodateien.Stellen Sie sich ein Videoformat als ZIP-Datei vor, die den codierten Video-Stream, die Tonspur und evtl. noch Untertitel enthält.

Bis jetzt existieren immer noch verschiedene Video- und Audio-Kodierungen. Obwohl die meisten Kameras und Mobiltelefone Filme im MP4-Format (mit H264-Codec) aufnehmen, gab es Vorbehalte, da der dort verwendete H264-Video-Codec unter Umständen zu Lizenzzahlungen verpflichtete. Aus diesen Gründen wurden für den Einsatz im Netz die freien Formate Ogg und WebM entwickelt. Mittlerweile hat sich WebM durchgesetzt und wird von allen Browser außer den IE9-11 unterstützt. Youtube, das ja zum Google-Konzern gehört, verwendet für alle Videos das webm-Format.

Example:

<video src="video.mp4" controls> </video>

Links

Anchor (Link)

Der HTML <a>-Tag ist ist gültig als Anker oder Hyperlink: Was zwischen öffnendem und schließendem a-Tag sitzt, wird zu einem klickbaren Link.

Example:

<a href="https://www.example.com">Visit Example.com</a>

Link

Das Tag <link> verbindet das HTML-Dokument mit externer Quelle. In der Regel wird es für die Einschaltung des externen Files, wo CSS-Stile und Schrifte eingegeben werden, verwendet. Das Tag <link> wird ungepaart verwendet, es enthält nur Attribute und wird immer innerhalb des Tages <head> (in der Regel vor dem Endtag </head>) platziert, dabei kann es mehrmals verwendet werden.

Example:

<link rel="stylesheet" href="styles.css">

Navigation (Nav)

Das <nav>-Tag ist für Links zu anderen Seiten und zu Bereichen innerhalb derselben Seite reserviert. Das <nav>-Element soll nur für die Haupt-Navigation der Seite benutzt werden und nicht für beiläufige Links einer Seite.

Example:

<nav> <a href="home.html">Home</a> <a href="about.html">About</a> <a href="contact.html">Contact</a> </nav>

Lists

Unordered List (ul)

Der <ul> Tag wird verwendet, um eine ungeordnete Liste zu erstellen, in der die Listenelemente mit Aufzählungspunkten dargestellt werden.

Example:

<ul> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul>

Ordered List (ol)

Der <ol> Tag wird verwendet, um eine geordnete Liste zu erstellen, in der die Listenelemente mit Nummern oder anderen geordneten Markierungen angezeigt werden.

Example:

<ol> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ol>

List Item (li)

Der <li> Tag wird verwendet, um ein Listenelement innerhalb einer ungeordneten oder geordneten Liste zu definieren.

Example:

<ul> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul>

Directory (dir)

Der <dir> Tag ist ein nicht standardisierter HTML-Tag, der zur Erstellung einer Verzeichnisliste verwendet wird, die eine spezielle Art von ungeordneter Liste ist.

Example:

<dir> <li>File 1</li> <li>File 2</li> <li>File 3</li> </dir>

Definition List (dl)

Das HTML- Element <dl> repräsentiert eine Beschreibungsliste. Das Element umschließt eine Liste von Begriffsgruppen (angegeben durch das <dt> -Element) und Beschreibungen (bereitgestellt durch <dd> -Elemente). Dieses Element wird häufig verwendet, um ein Glossar zu implementieren oder Metadaten (eine Liste von Schlüssel-Wert-Paaren) anzuzeigen.

Example:

<dl> <dt>Term 1</dt> <dd>Definition 1</dd> <dt>Term 2</dt> <dd>Definition 2</dd> </dl>

Definition Term (dt)

Der <dt> Tag wird verwendet, um einen Begriff innerhalb einer Definitionsliste zu definieren.

Example:

<dl> <dt>Term 1</dt> <dd>Definition 1</dd> <dt>Term 2</dt> <dd>Definition 2</dd> </dl>

Definition Description (dd)

Der <dd> Tag wird verwendet, um die Beschreibung oder Definition eines Begriffs innerhalb einer Definitionsliste festzulegen.

Example:

<dl> <dt>Term 1</dt> <dd>Definition 1</dd> <dt>Term 2</dt> <dd>Definition 2</dd> </dl>

Tables

Table (table)

Das Tabellenmodell von HTML gestattet Autoren die Anordnung von Daten – Text, vorformatierter Text, Bilder, Links, Formulare, Formularfelder, andere Tabellen und so weiter – in Zellen, die in Zeilen und Spalten angeordnet sind.

Jeder Tabelle kann eine Beschriftung zugewiesen werden (siehe das Element CAPTION), die eine kurze Beschreibung der Intention der Tabelle enthält. Zum Nutzen der Leute, die sprach- oder braille-basierte Benutzerprogramme verwenden, kann auch eine längere Beschreibung angegeben werden (über das Attribut summary).

Tabellenzeilen können in Kopf-, Fuß- und Rumpfbereiche geordnet werden (über die Elemente <THEAD>, <TFOOT> bzw. <TBODY>). Zeilengruppen vermitteln zusätzliche Strukturinformationen und können von Benutzerprogrammen auf eine Weise dargestellt werden, die diese Struktur unterstützt. Benutzerprogramme können die Bereiche Kopf, Rumpf und Fuß ausnutzen, um das von den Kopf- und Fußbereichen unabhängige Scrollen von Rumpf-Bereichen zu unterstützen. Werden lange Tabellen gedruckt, können die Kopf- und Fußinformationen auf jeder Seite, die Tabellendaten enthält, wiederholt werden.

Example:

<table> </table>

Caption (caption)

Der <caption> ag wird verwendet, um eine Beschriftung oder einen Titel für eine HTML-Tabelle anzugeben.

Example:

<table> <caption>Table Caption</caption> </table>

Table Header Cell (th)

Der <th> Tag wird verwendet, um Kopfzellen in einer HTML-Tabelle zu definieren.

Example:

<table> <tr> <th>Header 1</th> <th>Header 2</th> </tr> </table>

Table Row (tr)

Der <tr> Tag wird verwendet, um Zeilen in einer HTML-Tabelle zu definieren.

Example:

<table> <tr> <td>Row 1, Cell 1</td> <td>Row 1, Cell 2</td> </tr> </table>

Table Data Cell (td)

Der <td> Tag wird verwendet, um Datenzellen in einer HTML-Tabelle zu definieren.

Example:

<table> <tr> <td>Row 1, Cell 1</td> <td>Row 1, Cell 2</td> </tr> <tr> <td>Row 2, Cell 1</td> <td>Row 2, Cell 2</td> </tr> </table>

Table Head (thead)

Der <thead> Tag wird verwendet, um den Kopfzeileninhalt in einer HTML-Tabelle zu gruppieren.

Example:

<table> <thead> <tr> <th>Header 1</th> <th>Header 2</th> </tr> </thead> </table>

Table Body (tbody)

Der <tbody> Tag wird verwendet, um den Hauptinhalt in einer HTML-Tabelle zu gruppieren.

Example:

<table> <tbody> <tr> <td>Row 1, Cell 1</td> <td>Row 1, Cell 2</td> </tr> <tr> <td>Row 2, Cell 1</td> <td>Row 2, Cell 2</td> </tr> </tbody> </table>

Table Footer (tfoot)

<tfoot> -Tag für die Kombination von Fußzeile Inhalt HTML-Tabelle. <tfoot> -Element sollte sein <thead> und <tbody> jedes Teil (Footer, Header, Body) Elemente in Kombination verwendet , um die Tabelle zu definieren. Durch die Verwendung dieser Elemente, so dass der Browser die Fähigkeit hat, unabhängig von der Tabellenkopf und dem Tabellenfuß Tabelle Körperrolle zu unterstützen. Wenn enthalten mehrere sind lange Tabellen Seiten gedruckt, können die Tabelle Kopf- und Fußzeilen auf jeder Seite gedruckt werden, die Tabellendaten enthält. <tfoot> Tag muss in den folgenden Situationen verwendet werden: a <table> Kind-Elemente, erscheint nach dem <caption>, <colgroup> und <thead> -Element, bevor <tbody> und <tr> -Element.

Example:

<table> <tfoot> <tr> <td>Footer 1</td> <td>Footer 2</td> </tr> </tfoot> </table>

Table Column (col)

Das Tag <col> bestimmt die Eigenschaften einer oder mehreren Spalten in der HTML-Tabelle, die mit dem Tag <table> bestimmt wird. Das Tag <col> wird häufig mit dem Tag <colgroup>, das eine Gruppe mit den Gesamteigenschaften bestimmt, verwendet. Das Tag <col> wird ungepaart verwendet, ergo ist kein Endtag notwendig. In XHTML aber muss das Tag <col> mit dem Endtag (</col>) geschlossen werden. Das Tag <col> wird innerhalb des Tages <table>, vor den Tags <thead>, <tbody>, <tfoot> und <tr>; und nach dem Tag <caption> platziert, wenn es verwendet wird (im Tag <caption> fügen wir den Namen der Tabelle ein).

Example:

<table> <colgroup> <col span="2" style="background-color: #f2f2f2;"> </colgroup> <tr> <td>Row 1, Cell 1</td> <td>Row 1, Cell 2</td> </tr> <tr> <td>Row 2, Cell 1</td> <td>Row 2, Cell 2</td> </tr> </table>

Styles and Semantics

Style (style)

Der <style> Tag wird verwendet, um Inline-CSS-Stile zu definieren oder externe CSS-Dateien einzubinden.

Example:

<style> h1 { color: blue; font-size: 24px; } </style>

Div (div)

Die zwei HTML-Befehle <div> und <span> werden erst durch CSS extrem nützlich. Der HTML-Befehl <div> steht für das englische „division“ in der Bedeutung „Bereich“ und ist ein Block-Element (zur Erinnerung – Block-Elemente nutzen die komplette Browserfensterbreite). Lustigerweise hat der HTML-Befehl <span> im englischen auch die Bedeutung von „Bereich“, ist aber ein Inline-Element.

Beide Elemente werden primär zum Begrenzen von Bereichen genutzt. Über die Attribute der Elemente, kann dann Design in die umschlossenen Bereiche gebracht werden. Dabei können die Elemente geschachtelt werden.

Wichtig ist zu verstehen, was der Unterschied zwischen <div> und <span> ist:

<div> ist ein Block-Element. Ein <div> nimmt die komplette Breite ein, sofern nichts anderes in CSS umdefiniert wurde. <span> dagegen kann innerhalb eines Bereiches angewendet werden – <span> ist ein Inline-Element.

Example:

<div> <p>This is a div container.</p> <p>It can contain other elements.</p> </div>

Span (span)

HTML <span> separiert Inline-Elemente wie Textabschnitte, Eingabefelder oder Bilder für CSS oder Javascript. <span> ist nur ein Behälter ohne semantische Bedeutung und hat ohne CSS keine Auswirkung auf die Darstellung.

In dieser Hinsicht ist das <span>-Element das Gegenstück zum <div>-Element, das ein Blockelement bildet (CSS div führt zu einem Zeilenumbruch). Beide Elemente nehmen eine Reihe von weiteren Elementen auf, um Attribute oder CSS-Stile auf sie anzuwenden, ohne selber eine Formatierung einzubringen.

Example:

<p>This is a <span style="color: blue;">blue</span> text.</p>

Header (header)

Ein Header (deutsch: Kopf) ist der sichtbare Bereich im obersten Teil einer Webseite. Er ist ein fester Bestandteil im Website-Aufbau und bietet dem Nutzer grundlegende, wichtige Informationen. In der Regel enthält er neben dem Logo der Seite wichtige und häufig benutzte Navigationspunkte und -elemente wie „Login“ oder „Hilfe“. Ein Header ist domainübergreifend für alle Seiten gleich gestaltet. Er befindet sich über dem eigentlichen Hauptmenü oder schließt dieses mit ein. Aber wozu dient der Header einer Website? Eine wichtige Funktion dieses Websiteelements ist es, dem Besucher die Navigation, Orientierung und Interaktion innerhalb der Seite zu erleichtern. Übrigens: Der Header ist als Kopfzeile das Gegenstück zum Footer. So wird die Fußzeile bezeichnet, die sich ganz am Ende der Webseite befindet.

Example:

<header> <h1>Welcome to our website!</h1> <p>This is the header section.</p> </header>

Footer (footer)

<footer>-Tags können unter einzelnen Bereichen der Seite sitzen und den Abschluß der Webseite bilden, anders als das main-Element können footer-Elemente mehrfach innerhalb der Webseite verwendet werden.

<footer> – am Ende ist nicht Schluß Das <footer>-Element steht nicht nur für den endgültigen Fuß der Seite, sondern kann ebenso auch den Abschluss von Beiträgen in einer Blog-Seite bilden und zusätzliche Informationen zum Beitrag aufnehmen. Das können sowohl Copyright-Informationen als auch Informationen zum Autor oder Links zu weiteren Quellen sein. Typische Elemente, die mit einem <footer>-Element abgeschlossen werden können, sind z.B. <article>, <section> und <figure>. In einem <footer>-Element können beliebige Elemente (Text, Bilder und natürlich Links) liegen, nicht aber <footer>, <main> und <header>. Auf der anderen Seite sollte in <header>- und <address>-Elementen keine <footer>-Elemente liegen.

Example:

<footer> <p>This is the footer of the document.</p> </footer>

Main (main)

<main> stellt den Hauptinhalt der Webseite dar und sitzt dort, wo früher <div id="main">, <div id="page"> o.ä. saß. Das <main>-Element soll nur einmal im Dokument vorkommen und weder innerhalb eines <header>, <footer> oder <article>-Tags noch eines <section>-Tags sitzen.

Sectioning Tags – Gliederung der Webseite:

<main> gehört zu den Tags, die das Markup des Dokuments strukturieren: <article>, <section>, <header>, <footer>, <nav> und <aside>. Auch <main> wirkt gegen die Schwemme von <div>-Tags mit langen Listen von CSS-Klassen, die das Markup vor HTML5 in ein wildes und wirres Knäuel verwandelten.

Example:

<main> <h1>Main Content</h1> <p>This is the main section of the page.</p> </main>

Section (section)

Der Bereich, in dem die einzelnen Artikel untereinander angezeigt werden, wird in ein <div> gepackt. Hier gehört es auch hin, denn ein <div> ist ein neutrales Container-Element ohne spezielle Aufgabe, mit dem man mehrere Elemente zusammenfassen kann. Der Unterschied zwischen <article>, <div> und <section> In HTML5 gibt es noch ein Tag, das dem <article>-Element ähnelt. Dieses Tag heisst <section>. Einige Autoren verwenden nun <section> anstatt eines <div>, um mehrere <article>-Elemente auf einer Blog-Homepage zusammenzufassen. Das ist nicht ganz korrekt. Der Grund: Das <section>-Element ist nicht als Container gedacht. Diese Rolle behält nach wie vor das <div>. Eine <section> ist ein in sich geschlossenes Element, das z.B. eine Headline und einen Inhalt enthält. Als Faustregel kann man sich folgendes merken: Ein <div> ist immer dann die richtige Wahl, wenn man etwas zusammenfassen möchte. Ein <section>-Element steht für sich und hat immer eine eigene Headline bzw. könnte sinngemäß eine haben. Das <section>-Element steht damit quasi eine Stufe tiefer als das <article>-Element, das relativ komplex strukturierte Inhalte aufnehmen kann und ausdrücklich für Blog-Posts konzipiert wurde.

Example:

<section> <h2>Section Heading</h2> <p>This is a section of the page.</p> </section>

Article (article)

Das HTML- Element <article> stellt eine in sich geschlossene Komposition in einem Dokument, einer Seite, einer Anwendung oder einer Site dar, die unabhängig vertreibbar oder wiederverwendbar sein soll (z. B. in Syndikation). Beispiele hierfür sind: ein Forumsbeitrag, ein Zeitschriften- oder Zeitungsartikel oder ein Blogeintrag, eine Produktkarte, ein vom Benutzer eingereichter Kommentar, ein interaktives Widget oder Gadget oder jedes andere unabhängige Inhaltselement. Ein gegebenes Dokument kann mehrere Artikel enthalten; zum Beispiel in einem Blog, das den Text jedes Artikels nacheinander anzeigt, während der Leser scrollt, wäre jeder Beitrag in einem <article> -Element enthalten, möglicherweise mit einem oder mehreren <section> s darin.

Example:

<article> <h2>Article Title</h2> <p>This is an article.</p> </article>

Aside (aside)

Das <aside> Element wurde mit HTML5 neu eingeführt und kann auf zwei Arten eingesetzt werden. Grundsätzlich basiert der Kontext des <aside> Elements darauf ob es innerhalb oder außerhalb eines article Elementes eingesetzt wird.

In Magazinen sieht man oft Info-Kästen die aus dem eigentlichen Inhalt des Artikels herausgezogen sind und z.B. etwas besonders relevantes aus dem Artikel noch einmal gesondert hervorheben. Wenn das der Einsatzzweck des aside Elementes ist, so muss es innerhalb des article Elementes platziert werden, da es eine Beziehung zum Inhalt des Artikels hat.

Inhalt der mit dem aktuellen Artikel, Blogpost usw. in Beziehung steht

Example:

<aside> <h3>Sidebar</h3> <p>This is a sidebar section.</p> </aside>

Details (details)

Der <details> Tag wird verwendet, um ein Offenlegungs-Widget zu erstellen, das umgeschaltet werden kann, um zusätzliche Inhalte ein- oder auszublenden. Es wird normalerweise verwendet, um erweiterbare Abschnitte mit einer zusammenfassenden Überschrift zu erstellen.

Example:

<details> <summary>Click me to toggle the details</summary> <p>Hidden details content.</p> </details>

Dialog (dialog)

Der <dialog> Tag wird verwendet, um ein modales oder ein Dialogfeld innerhalb einer Webseite zu erstellen. Es wird in der Regel für interaktive Eingabeaufforderungen oder Nachrichten verwendet.

Example:

<dialog open> <p>This is a dialog box.</p> <button>Close</button> </dialog>

Summary (summary)

Der <summary> Tag wird verwendet, um eine sichtbare Überschrift oder Zusammenfassung für das <details>-Element zu definieren. Es wird angeklickt, um die Sichtbarkeit des Inhalts der Details umzuschalten.

Example:

<details> <summary>Click me to toggle the details</summary> <p>Hidden details content.</p> </details>

Data (data)

Mit <data>-Attributen (Custom Data Attributes) haben Sie die Möglichkeit, Elementen eigene Attribute mitzugeben, die dann per Script ausgewertet oder mit CSS genutzt werden können. In diesen Attributen können Sie Informationen, die nicht visuell präsentiert werden sollen, zur Verfügung stellen.

Anders als Mikroformate sind <data>-Attribute interne Angaben und werden von Suchmaschinen nicht ausgewertet. Sie können beliebig viele <data>-Attribute definieren; diese müssen nur mit <data>- beginnen und aus Kleinbuchstaben bestehen.

Example:

<p>The average temperature is <data value="23">23</data>°C.</p>

Meta Info

Head (head)

Der <head> Tag wird verwendet, um den Kopfbereich eines HTML-Dokuments zu definieren. Er enthält Metainformationen über das Dokument, wie z. B. den Titel, Links zu externen Stylesheets und Skripten sowie andere Metadaten.

Example:

<head> <title>My Web Page</title> <meta charset="UTF-8"> <link rel="stylesheet" href="styles.css"> <script src="script.js"></script> </head>

Meta (meta)

Das Tag <meta> enthält Metadaten, die dem Browser und der Suchmaschine über die Einzelheiten des HTML-Dokuments (Codierung des Dokuments, Angaben des Autors, Schlüsselwörter, Beschreibung von Webseiten usw.) informieren. Die Metadaten werden auf der Webseite nicht angezeigt. Das Tag <meta> wird im Tag <head> platziert. In einem Dokument kann man einige Tags <meta> verwenden.

Example:

<meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="This is my website">

Base (base)

Der <base> Tag gibt die Basis-URL und das Ziel für alle relativen URLs in einem Dokument an. Er wird innerhalb des Kopfbereichs verwendet und hilft, relative Links korrekt aufzulösen.

Example:

<base href="https://www.example.com/" target="_blank">

Basefont (basefont)

Der <basefont> Tag wird verwendet, um die Basisschriftgröße, -farbe und -schriftart für ein Dokument festzulegen. Es ist jedoch in HTML5 veraltet, und seine Verwendung wird nicht empfohlen. Stattdessen sollte CSS für die Gestaltung des Dokuments verwendet werden.

Example:

<basefont size="4" color="red" face="Arial">

Programming

Script (script)

Das HTML <script>-Element lädt Scripte innerhalb des <head>- und <body>-Elementen einer HTML-Seite. <script> wird in erster Linie zum Einbinden von Javascript in Webseiten benutzt. Das <script>-Element darf sowohl im head- also auch an jeder Stelle im <body>-Element stehen.

Im <script>-Element können Script-Anweisungen direkt eingesetzt werden, aber <script> kann auch eine externe Script-Datei mittels <scr>-Attribut laden. Ohne weitere Vorsorge führt der Browser führt das Script sofort an Ort und Stelle aus.

Javascript ändert Inhalte, CSS-Stile oder HTML-Elemente und kann erst fehlerfrei ausgeführt werden, wenn das DOM, CSS-Dateien, globale CSS-Stile und der Inhalt bereits geladen sind. Wenn das Javascript im <head> der HTML-Seite geladen wird, stellt der Browser alle weiteren Elemente zurück, bis das Script geladen und interpretiert ist. Das kann zu einem sichtbaren Aussetzer beim Laden der Seite führen. Wenn das Script schon in HTML <head> geladen werden muss, dann besser nach allen CSS-Dateien.

Example:

<script src="script.js"></script> <script> // JavaScript code goes here </script>

Noscript (noscript)

Der <noscript> Tag wird verwendet, um alternative Inhalte anzuzeigen, wenn ein Browser kein JavaScript unterstützt oder JavaScript deaktiviert hat.

Example:

<noscript> <p>JavaScript is disabled in your browser. Please enable it to view this website.</p> </noscript>

Applet (applet)

Der <applet> Tag wird verwendet, um Java-Applets in ein HTML-Dokument einzubetten. Allerdings werden Java-Applets von den meisten modernen Browsern nicht mehr unterstützt.

Example:

<applet code="MyApplet.class" width="300" height="200"> Applet content goes here </applet>

Embed (embed)

Das Tag <embed> wird als Container für die externen Anwendungen, den multimedialen und interaktiven Inhalt, die der Browser nicht versteht, verwendet. Für ihre korrekte Darstellung muss man externes Plug-in oder spezielle Programme anschließen. Wie der beigefügte Inhalt angezeigt wird, hängt vom Typ der Datei, von den Attributen des Tages <embed> sowie von Plug-ins, die im Browser installiert sind, ab. Obwohl das Tag <embed> von den Browsern unterstützt wurde, war es in HTML 4 nicht eingeschlossen. Fürs Hochladen von Objekten in HTML 4 wird das Tag <object> verwendet. Da nicht alle Browser in sich befindende Information korrekt darstellen, wird für die Unterstützung der großen Anzahl von Browsern sowie die Validität des Dokumentes das Tag <embed> innerhalb des Tages <object> verwendet. In HTML 5 gibt es kein Problem mit der Validität des Dokuments, weil das Tag <embed> bereits ein ofizielles Status in der neuen Version von HTML hat.

Example:

<embed src="video.mp4" type="video/mp4" width="640" height="480">

Object (object)

Das <object> Tag ist ein HTML-Tag und wird verwendet, um Multimedia wie Audios, Videos, Bilder, PDFs und Flash in Webseiten anzuzeigen. Es kann auch zur Anzeige einer anderen Webseite innerhalb der HTML-Seite verwendet werden.

Der <param> Tag wird auch zusammen mit diesem Tag verwendet, um verschiedene Parameter zu definieren. Jeder Text, der innerhalb von <object> und </object> Tags geschrieben wird, wird als alternativer Text betrachtet, der erscheint, wenn die angegebenen Daten vom Browser nicht unterstützt werden.

Dieser Tag unterstützt alle Global- und Ereignisattribute von HTML..

Example:

<object data="file.pdf" type="application/pdf" width="500" height="600"> Alternative content goes here </object>

Param (param)

Der <param> Tag wird verwendet, um Parameter für Objekte oder Applets zu definieren, die in ein HTML-Dokument mit der <object> or <applet> Tag benutzt werden.

Example:

<object data="file.swf"> <param name="autoplay" value="true"> <param name="loop" value="false"> </object>

Formatting

Abbreviation (abbr)

<abbr> kennzeichnet einen Text als Abkürzung eines längeren Wortes oder Ausdrucks oder Akronyms (Abkürzung aus den Anfangsbuchstaben). Die Browser stellen Text in abbr durch eine gestrichelte Linie unter dem Text heraus und machen die Erklärung mit einer kleinen Verzögerung beim Hovern mit der Maus sichtbar.

Abkürzungen sind sinnvoll für den Besucher: Wer will schon dauernd Cascading Style Sheets statt einfach CSS lesen? Die Erklärung einer Abkürzung bringt zusätzliche Informationen für Browser, Übersetzungen und auch für die Suchmaschinen.

<abbr> – abbreviation – Kürzel

Die meisten Browser markieren den Text innerhalb eines abbr-Tags durch einen Unterstrich, aber eine zuverlässige Kennzeichnung gibt es nur durch eine CSS-Eigenschaft für <abbr>.

HTML abbr ist ein Phrase-Tag (Inlineelement), d.h., es führt nicht zu einem Zeilenvorschub und kann in jedem Block- und Inline-Element stehen. Phrase-Tags wie code und cite umschließen eine oder mehrere Wörter innerhalb eines Satzes.

Example:

<p>I work at <abbr title="OpenAI">AI</abbr> company.</p>

Template (template)

Das Element <template> verwendet man für die Aufbewahrung der Vorlagen des HTML-Kodes, die man klonen und ins Dokument mithilfe von Skripten platzieren kann. Es wird gewöhnlich für die Elemente mit einer wiederholenden Struktur wie Listen, Tabellen usw. verwendet.

Das Tag <template> unterscheidet sich von anderen Tags dadurch, dass sein Inhalt besonders bearbeitet wird. Die Fragmente des Kodes werden auf der Seite des Kunden bewahrt und werden während des Hochladens der Seite nicht angezeigt. Der Inhalt wird nur dann aktiviert (das Video wird gespielt, die Skripte werden durchgeführt usw.), wenn er im Dokument platziert wird.

Der Browser bearbeitet den Inhalt des Tages, um die Validität des Codes zu bestätigen.

Das Tag <template> kann tatsächlich an einer beliebigen Stelle des HTML-Dokumentes platziert werden sein: <head>, <body> oder <frameset> und kann jede Art von Inhalt, der in diesen Elementen erlaubt ist, enthalten.

Das Tag <template> ist ein neues Element in HTML 5
